    Create `frontend/src/components/PostProcessEngine.tsx` as a React component that accepts `viewer: Viewer | null` as a prop. It renders null to the DOM and manages the entire post-processing stage lifecycle.

    **Stage creation (inside `useEffect(() => {}, [viewer])`):**
    - Guard double-init with `const initRef = useRef(false)`. At top of effect: `if (!viewer || initRef.current) return; initRef.current = true;`
    - Store all stages in `const stagesRef = useRef<Record<string, PostProcessStage | PostProcessStageComposite | object>>({});`
    - Create a `const uniformsRef = useRef<PostProcessUniforms>({ bloomIntensity: 0.5, sharpenAmount: 0.5, gainAmount: 1.0, scanlineSpacing: 3, pixelationLevel: 1 });`

    **NVG stage:** `PostProcessStageLibrary.createNightVisionStage()` — no uniforms needed
    **Noir stage:** `PostProcessStageLibrary.createBlackAndWhiteStage()` — no uniforms needed
    **FLIR stage:** Custom `PostProcessStage` with fragment shader implementing luminance-to-iron-gradient mapping (black→blue→red→yellow→white). Use the exact GLSL from the research file. No slider-driven uniforms needed for FLIR.
    **CRT stage:** `PostProcessStageComposite` with two passes:
      - Pass 1 (scanlines): `uniform float u_scanlineSpacing; // () => uniformsRef.current.scanlineSpacing` — divide screen by spacing, draw dark bands at integer intervals
      - Pass 2 (barrel): slight radial distortion, chromatic aberration on RGB channels (±1px shift at edges)
    **Sharpen/Gain/Bloom stages:** Three separate `PostProcessStage` instances for the independent slider-driven effects:
      - `sharpStage`: convolution kernel sharpening, `uniforms: { u_amount: () => uniformsRef.current.sharpenAmount }`
      - `gainStage`: `vec4(color.rgb * u_gain, color.a)`, `uniforms: { u_gain: () => uniformsRef.current.gainAmount }`
      - `pixelStage`: pixel-block averaging at `u_pixels = floor(uv * u_level) / u_level`, `uniforms: { u_level: () => uniformsRef.current.pixelationLevel }`
    - Bloom is a property of the collection: `viewer.scene.postProcessStages.bloom` — always available, no add() required. Read `bloomIntensity` via `uniformsRef` by attaching a `viewer.scene.preRender` listener that sets `bloom.uniforms.contrast` from `uniformsRef.current.bloomIntensity * 128`.

    **Add all custom stages to the collection (all disabled):**
    ```typescript
    [stagesRef.current.nvg, stagesRef.current.noir, stagesRef.current.flir,
     stagesRef.current.crt, stagesRef.current.sharp, stagesRef.current.gain,
     stagesRef.current.pixel].forEach(s => {
       (s as PostProcessStage).enabled = false;
       viewer.scene.postProcessStages.add(s as PostProcessStage);
    });
    ```

    **Cleanup:** Return a cleanup from the effect that removes all added stages via `viewer.scene.postProcessStages.remove(stage)` for each custom stage in stagesRef. Remove the preRender listener.

    **Preset effect (separate `useEffect([visualPreset])`):**
    - Disable all custom stages
    - Enable only the stages appropriate for the active preset:
      - `'normal'`: all disabled, bloom disabled
      - `'nvg'`: nvg enabled
      - `'crt'`: crt enabled
      - `'flir'`: flir enabled
      - `'noir'`: noir enabled

    **Uniforms sync effect (separate `useEffect([postProcessUniforms])`):**
    - Write Zustand uniforms into `uniformsRef.current` so function-style uniforms pick up the new values on the next frame: `uniformsRef.current = { ...uniformsRef.current, ...postProcessUniforms }`
    - For bloom: also set `viewer.scene.postProcessStages.bloom.uniforms.contrast` directly here (in addition to the preRender listener)

    Component consumes `visualPreset` and `postProcessUniforms` from `useAppStore()`.

    **TypeScript:** Import `PostProcessStageLibrary, PostProcessStage, PostProcessStageComposite` from `'cesium'`.
