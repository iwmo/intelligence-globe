    **Step 3 — Create backend/app/workers/ingest_ais.py:**

    Export three items:

    1. `parse_ais_message(msg: dict) -> dict | None` — pure function (unit-testable, no Redis/DB):
       - Returns None if msg.get("MessageType") != "PositionReport"
       - Extracts MetaData and Message.PositionReport
       - Returns None if MetaData.latitude or MetaData.longitude is None
       - Returns dict: {mmsi, ship_name, latitude, longitude, sog, cog, true_heading, nav_status, time_utc}
       - true_heading: `pos.get("TrueHeading", 511)` — 511 means not available
       - mmsi: `str(meta["MMSI"])` — always store as string

    2. `async def batch_flush_ships_to_pg(redis_client, session_factory) -> int`:
       - Scans Redis for all keys matching `ship:*`
       - For each key, hgetall to get the ship dict
       - Batch-upserts to PostgreSQL via pg_insert(Ship).on_conflict_do_update(index_elements=["mmsi"])
       - Returns count of upserted ships

    3. `async def run_ais_worker() -> None` — long-lived worker entry point:
       - Creates redis.asyncio client from REDIS_URL env var
       - Builds subscribe_msg with AISSTREAM_API_KEY from env, global bounding box, FilterMessageTypes=["PositionReport"]
       - `async for websocket in connect(WS_URL, ping_interval=None):` outer reconnect loop
       - FIRST action after connection: `await websocket.send(subscribe_msg)` — no sleep before this
       - `async for raw in websocket:` inner message loop
       - Call parse_ais_message(); skip None; HSET to Redis with 600s TTL
       - Run batch_flush_ships_to_pg every BATCH_INTERVAL=30 seconds using asyncio.create_task scheduling
       - `except Exception: continue` for reconnect

    **Module-level entry point** so `python -m app.workers.ingest_ais` works:
    ```python
    if __name__ == "__main__":
        asyncio.run(run_ais_worker())
    ```
