Cesium LabelCollection API:
```typescript
import { LabelCollection, LabelStyle, VerticalOrigin, HorizontalOrigin } from 'cesium';

const labelCollection = viewer.scene.primitives.add(new LabelCollection());
const lbl = labelCollection.add({
  position: someCartesian3,
  text: 'STARLINK-1234',
  font: '11px monospace',
  fillColor: Color.fromCssColorString('#00D4FF'),
  outlineColor: Color.BLACK,
  outlineWidth: 2,
  style: LabelStyle.FILL_AND_OUTLINE,
  pixelOffset: new Cartesian2(0, -18),  // offset above the point/billboard
  verticalOrigin: VerticalOrigin.BOTTOM,
  horizontalOrigin: HorizontalOrigin.CENTER,

  show: false,  // start hidden; visibility controlled by showEntityLabels
});
```

CRITICAL constraints:
- Satellite layer: GPU budget is tight (5,000+ points). LabelCollection is a SEPARATE primitive from PointPrimitiveCollection — not embedded in it. Create labelCollectionRef alongside collectionRef.
- Aircraft layer: Labels parallel to BillboardCollection. Module-scope Map `labelsByIcao24: Map<string, any>` mirrors `billboardsByIcao24`. Cleanup must also clear `labelsByIcao24`.
- scaleByDistance for satellites: `new NearFarScalar(5e5, 1.2, 5e7, 0.0)` — labels disappear at global altitude (consistent with point scaleByDistance already in use).
- scaleByDistance for aircraft: `new NearFarScalar(1e4, 1.4, 5e6, 0.0)` — match billboard scaleByDistance range.
- pixelOffset: use `new Cartesian2(0, -18)` to position label above the entity icon.
- `useSettingsStore` is imported from `../store/useSettingsStore` (not useAppStore).
