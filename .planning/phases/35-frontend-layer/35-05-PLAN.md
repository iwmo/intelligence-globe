<!-- OsintEventPanel prefill useEffect (add at top of component, after useState declarations) -->
```typescript
const prefill = useAppStore(s => s.gdeltOsintPrefill);
useEffect(() => {
  if (prefill) {
    setLat(String(prefill.lat));
    setLon(String(prefill.lon));
    setTs(prefill.ts.slice(0, 16));          // datetime-local format: YYYY-MM-DDTHH:mm
    setSourceUrl(prefill.sourceUrl ?? '');
    useAppStore.getState().setGdeltOsintPrefill(null); // clear immediately to prevent re-trigger
  }
}, [prefill]);
```

Pitfall to avoid: if OsintEventPanel calls `setGdeltOsintPrefill(null)` in the useEffect, App.tsx re-evaluates `open` as `false || null !== null = false`, which would close the panel. To prevent this, App.tsx must also gate on `osintPanelOpen`. The sequence is: prefill set → App.tsx opens panel (open=true because prefill !== null) → OsintEventPanel mounts and clears prefill → open now gated only by osintPanelOpen. But osintPanelOpen is still false at this point, which would close the panel immediately. Fix: set `osintPanelOpen = true` in App.tsx when `gdeltOsintPrefill !== null` via a separate useEffect, then clear prefill in OsintEventPanel. Or: OsintEventPanel clears prefill AFTER setting form state, and App.tsx sets osintPanelOpen as a side-effect.

Cleaner fix: In App.tsx, add a `useEffect` that sets `osintPanelOpen(true)` when `gdeltOsintPrefill` becomes non-null:
```typescript
useEffect(() => {
  if (gdeltOsintPrefill !== null) setOsintPanelOpen(true);
}, [gdeltOsintPrefill]);
```
Then OsintEventPanel can safely clear prefill without closing itself (osintPanelOpen is now true). The `onClose` handler clears both `osintPanelOpen` and `gdeltOsintPrefill`.
