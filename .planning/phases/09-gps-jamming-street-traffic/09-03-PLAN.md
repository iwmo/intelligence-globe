  <action>
    Create frontend/src/hooks/useStreetTraffic.ts. This hook manages road geometry state — not a React Query hook (no API route needed). Uses useState + useEffect.

    Interface:
    ```typescript
    export interface RoadSegment {
      coordinates: [number, number][]; // [lon, lat] pairs
    }

    export interface StreetTrafficState {
      roads: RoadSegment[] | null;
      isLoading: boolean;
      error: string | null;
    }
    ```

    Hook signature: `export function useStreetTraffic(viewer: Viewer | null): StreetTrafficState`

    Implementation:
    1. State: roads (RoadSegment[] | null), isLoading (boolean), error (string | null)
    2. useEffect on viewer ref — subscribes to viewer.camera.moveEnd event
    3. On moveEnd:
       a. Get altitude: viewer.camera.positionCartographic.height
       b. If altitude > 500_000: setRoads(null) — layer invisible, no fetch
       c. If altitude > 100_000 (but ≤ 500 km): setRoads(null) — layer visible gate but no road fetch (avoid Overpass timeout on wide viewports per RESEARCH.md Pitfall 3)
       d. If altitude ≤ 100_000: compute viewport bbox via viewer.camera.computeViewRectangle(), call fetchRoadsForViewport(bbox)
    4. fetchRoadsForViewport(bbox): POST to https://overpass-api.de/api/interpreter with highway query. Parse response — do NOT import osmtogeojson yet (that npm install happens in Plan 05). Instead, use a minimal inline parser:
       - Parse Overpass JSON: data.elements where element.type === 'way' and element.geometry exists
       - Each way.geometry is an array of {lat, lon} nodes → convert to [lon, lat][] coordinates
       - Return as RoadSegment[]
    5. Cleanup: remove moveEnd event listener on effect cleanup
    6. Debounce: only fetch if at least 2 seconds have passed since last fetch (avoid rapid re-fetch on smooth camera movement). Use a ref-stored timestamp.

    Error handling: catch fetch errors, set error state, do not crash.

    OVERPASS query (use highway types from RESEARCH.md):
    `[out:json][timeout:25];\nway["highway"~"^(motorway|trunk|primary|secondary|tertiary|residential)$"](${south},${west},${north},${east});\nout geom;`

    Do NOT use osmtogeojson import — inline parser only. This avoids build error before Plan 05 npm install.
