    Step 3 — Create backend/app/api/deps.py:

    ```python
    """
    FastAPI shared dependencies.

    verify_api_key: protects write endpoints with a static API key.
    Apply to routes via dependencies=[Depends(verify_api_key)].
    Future write routes should import this function rather than duplicating logic.
    """
    from fastapi import Header, HTTPException

    from app.config import settings


    async def verify_api_key(x_api_key: str = Header(...)) -> None:
        """Raise HTTP 401 when X-API-Key header is missing or does not match API_KEY.

        Note: FastAPI raises 422 automatically when Header(...) is present but the
        header is absent entirely — this is acceptable behaviour; 422 and 401 both
        reject the request. The three auth tests cover: no header (401 or 422 — test
        checks 401 which tests the case after the route is wired), wrong value (401),
        correct value (201).

        Timing-safe comparison is not used here: the threat model is a single-user
        homelab tool with no exposure to timing attacks. == is sufficient.
        """
        if x_api_key != settings.api_key:
            raise HTTPException(status_code=401, detail="Invalid or missing API key")
    ```

    IMPORTANT: The parameter must be named `x_api_key` (with the `x_` prefix) so
    FastAPI maps it to the `X-Api-Key` HTTP header. A parameter named `api_key` would
    map to the `Api-Key` header — wrong, always 422.
