CRITICAL codebase constraints (from STATE.md):
- SVG icon canvases pre-rendered at module scope (not per-entity) — pre-rendered AIRCRAFT_ICON must NOT change
- Billboard `rotation` is a single float in radians — Cesium's Billboard.rotation rotates the 2D icon in screen space around the billboard centre
- `alignedAxis: Cartesian3.ZERO` means rotation is screen-space (2D), not world-space 3D bank — this is the correct plane for a top-down icon visual banking effect
- Roll angle from ADSB.lol: positive = right bank, negative = left bank (standard aviation convention)

Rotation formula:
  `rotation = CesiumMath.toRadians(-(true_track ?? 0) + (roll ?? 0))`
  Explanation: heading points the nose; roll tilts the wings. In screen-space top-down view, adding roll degrees to the rotation rotates the icon, visually banking the wings. Positive roll (right bank) should tilt the right wing down — in Cesium screen-space the rotation axis points out of screen, so adding roll directly achieves the correct visual. Negative roll tilts left correctly.

Test pattern (follow AircraftLayer.debounce.test.tsx pattern):
- Unit-test the rotation formula as a pure function — no Cesium mock needed
- Extract rotation calculation into a named helper: `computeIconRotation(trueTrack: number | null, roll: number | null): number` (returns radians)
- Tests verify the helper output for various input combinations
