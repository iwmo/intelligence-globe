**Quick Start section (`## Quick Start`):**
Numbered list with exact, copy-pasteable commands. Do NOT use vague instructions.

```
1. git clone https://github.com/<your-username>/opensignal-globe.git
   cd opensignal-globe

2. cp .env.example .env

3. Edit .env — fill in the five required values (see "API Keys" below).
   POSTGRES_PASSWORD is pre-set to `changeme` and works for local use — do not delete it.

4. docker compose up

5. Open http://localhost in your browser.
```

Note: `VITE_CESIUM_ION_TOKEN` is required — the build fails without it. Cesium Ion has a free tier.

**API Keys section (`## API Keys`):**
Markdown table with columns: Variable(s) | Service | Free Tier | Where to Register

Rows:
| `OPENSKY_CLIENT_ID` + `OPENSKY_CLIENT_SECRET` | OpenSky Network (live aircraft positions) | Yes | https://opensky-network.org/ |
| `AISSTREAM_API_KEY` | AISStream.io (maritime AIS data) | Yes | https://aisstream.io/ |
| `VITE_CESIUM_ION_TOKEN` | Cesium Ion (3D globe rendering) | Yes | https://ion.cesium.com/tokens |
| `API_KEY` | Internal write-endpoint authentication | N/A — set any secret string | — |

Follow the table with a note: all four key types are required. The stack will not start if `VITE_CESIUM_ION_TOKEN` or `POSTGRES_PASSWORD` is missing.

**Architecture section (`## Architecture`):**
Brief description of the service topology, accurate to Phase 29 output:

```
Browser → http://localhost:80
              ↓
        nginx (frontend container)
              ↓  /api/* proxy
        FastAPI backend :8000 (internal)
              ↓
        PostgreSQL/PostGIS + Redis
              ↓
        worker + ais-worker (background ingestors)
```

Services started by `docker compose up`:
- `postgres` — PostGIS 16-3.5
- `redis` — Redis 7 alpine
- `backend` — FastAPI (uvicorn), internal port 8000
- `worker` — background data processor
- `ais-worker` — AIS maritime data ingestor
- `frontend` — nginx serving the production React/CesiumJS build + reverse proxy

**CI section (`## CI`):**
One short paragraph: GitHub Actions runs four checks on every push and PR — pytest (backend), vitest + tsc (frontend), gitleaks (secret scanning), docker build (both images). All four must pass before merge.

**Security Note section (`## Security Note`):**
If you fork this repository or plan to make it public, rotate the following credentials before doing so — earlier versions of the project may have contained real values in git history:
- OpenSky OAuth2 client secret (`OPENSKY_CLIENT_SECRET`)
- AISStream API key (`AISSTREAM_API_KEY`)
Use `git filter-repo` to purge history or reset the repository.

**License section (`## License`):**
One line: `MIT — see [LICENSE](LICENSE)`
